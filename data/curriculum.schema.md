# 课程数据 schema(单一真相源)

> 三个视图都只是这份数据的不同渲染。**新增知识点 = 往数据里加一条;优化沉淀 = 改某条的字段。**
> 对应「项目说明」第 5.1 节。类型定义见 [`src/data/types.ts`](../src/data/types.ts)。

## 脉(vein)

每个知识点归属一条「脉」。脉定义(`id → { name, color }`)集中在 [`src/data/veins.ts`](../src/data/veins.ts)。

| id | name | color |
|---|---|---|
| `shu` | 数 · 十进制 | `#d9b65a` |
| `jj` | 加与减 | `#e07a55` |
| `bi` | 比较 | `#6fb7ad` |
| `fl` | 分类与位置 | `#8aa6d8` |
| `tu` | 图形 | `#8fbf86` |
| `rmb` | 人民币 · 元角分 | `#c98bb5` |

## 境界(tier)

技能树分层用。索引 0..6:

`["起手","筑基","小成","登堂","入室","大成","出师"]`

## 知识点(node)字段

| 字段 | 类型 | 必填 | 说明 | 用于 |
|---|---|---|---|---|
| `id` | string | ✓ | 唯一标识(前置引用靠它) | 全部 |
| `name` | string | ✓ | 知识点名 | 全部 |
| `vein` | VeinId | ✓ | 所属脉 | 树形(归属)、配色 |
| `tier` | 0..6 | ✓ | 境界层级 | 技能树(分层) |
| `vol` | "上" \| "下" | ✓ | 上/下册 | 标注 |
| `truth` | string | ✓ | 真身一句话(深层结构) | 全部(悬停读出) |
| `forms` | string[] | ✓ | 化形/常见题型 | 树形叶子、出题者任务 |
| `req` | string[] | ✓ | 前置知识点 id(**可跨脉**) | 技能树(解锁/连线) |
| `from` | string | — | 缘起/叙事(它从哪长出来) | 文案,可选 |
| `note` | string | — | 编者视角(为什么这么教) | 文案,可选 |

## 视图各取所需

- **生命之树(TreeView)**:用 `vein`(归属)+ `forms`。
- **修炼路线(SkillTreeView)**:用 `tier` + `req`。
- **真身谱(AtlasView)**:用 `vein`/`truth` + `forms`。

三者读同一批记录。新增学科只需提供一份同构的 `*.json`,视图即可复用。
